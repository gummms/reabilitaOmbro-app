# REABILITA_OMBRO_SPECIFICATIONS

## 1. VISION
Create an application that guides users through the rehabilitation of shoulder injuries, following the exercises and protocols defined in the `./squad/assets/CLIENT_MOCKUP.md`. The main flow of the app is: `a: Login/Register``b: Home` -> `c: Exercise` -> `d: Feedback` -> `b: Home`.

## 1.1. SCREEN FLOW
- **Screen A**: Login/Register
- **Screen B**: Home: "welcome message", "Fase 1", "Fase 2", "Fase 3";
- **Screen C**: Exercise: "exercise video", "next exercise", "previous exercise", "timer of the same video length";
- **Screen D**: Feedback: after all exercises: "feedback message", "finish button";

## 2. UI/UX STANDARDS
- **Styling**: Strictly use standard Flutter layout widgets (`Padding`, `SizedBox`, `8-point grid`). Do not attempt to use HTML/CSS or Tailwind utility strings.
- **Responsiveness**: Use Flutter's `LayoutBuilder` or `MediaQuery` to adapt the dashboard for larger screens (tablets).
- **Design System**: Refer to default components and best practices in `./squad/assets/componentes_padrao`.
